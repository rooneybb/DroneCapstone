1. Download ApacheTomcat (used newest version) to C drive
2. Download Jenkins WarFile (used newest version)
3. Place Jenkins WarFile in \webapps directory of ApacheTomcat
4. Startup ApacheTomcat from \bin directory by running startup.bat
5. Default Jenkins port is 8080, access by going to localhost:8080/jenkins
5a. Create Inbound rule on firewall opening up port 8080 to publicly expose jenkins server, ip:8080/jenkins
6. Go to Manage Jenkins, then to Manage Plugins
7. Select Available Tab, search for necessary plugins, Med-Fleet used:
7a. Github plugin
--Need to set up link to SAREC-Lab/SEStudio-MedFleet Git Repo
--Set up repo polling at specified interval, will look for changes, and start specified jobs if diff is found
7b. Android Emulator Plugin
--Poll for any changes to PhoneApp folder, pulls code, grabs gradlew file, opens Android emulator and verifies build with each check-in 7c. HTTP POST Plugin
-- Send post requests to specified sites, in our case ticketing system
7d. SiteMonitor Plugin
-- Monitor any ip, used to lightly monitor all hosts to ensure they have heartbeat
7e. Email Extension Plugin (Never got fully working, gmail would block login attmepts, but can tell something is wrong anytime security message is received from google!)
--Supposed to leverage gmail instead of needing smtp server, to send notifications of failure or no heartbeat, see above
7f. Maven Plugin (never got fully working)
--Poll for any changes to MissionControl folder, pulls code, grabs pom file, and verifies build with each check-in
